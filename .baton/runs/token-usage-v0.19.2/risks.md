# Risks — token-usage-v0.19.2

| 위험 | 영향 | 완화 |
|---|---|---|
| 추정치를 실측으로 오인 | 잘못된 비용 인지 | `estimated` 플래그 + 표시에 "추정/실측" 병기 + anyEstimated 주석. 실제 디스패치 시 실측 자동 대체. |
| 추정 정확도 낮음 | 수치 신뢰도 | heuristic(chars/4) 근사임을 코드 주석·표시에 명시. 의사결정용 아닌 참고용. |
| 구독 잔량 기대 | "남은 양" 안 보임 불만 | 설계상 명시적 비목표(공식 API 없음, credential 금지). "Baton 호출분"만임을 표시에 명확화. |
| 스키마 변경 회귀 | team-run/봉투 깨짐 | `usage` **선택 필드** → 회귀 0(테스트). |
| 실측 포맷 불확실(codex) | 미래 파싱 불일치 | 이번엔 추정 + `metadata.usage` 수신 **계약만** 정의. 실제 파싱은 디스패치 마일스톤에서 어댑터별 검증. |

## 비목표 (재확인)
실제 codex/claude usage 파싱, 구독 플랜 잔량/계정 토큰 접근, 예산 설정/초과 게이트, USD 비용
환산, Swift 표시.

## 후속 (로드맵)
- **실제 디스패치**: 어댑터가 provider usage를 `metadata.usage`로 정규화(예: Claude
  `--output-format json`) → 같은 필드에 실측(estimated:false) 자동 반영.
- 예산 설정(플랫폼별 한도) + 초과 경고/중단 게이트("남은 = 한도 − 사용").
- USD 환산(단가 테이블), Swift 모니터 사용량 표시.
