# Baton macOS UX

이 문서는 macOS 앱의 디자인 언어와 수동 QA 기준을 기록한다.

## 디자인 언어

- 테마는 다크 고정이다. 기본 배경은 `#141414`, 표면은 `#1F1F1F`와 `#242424`를 사용한다.
- 텍스트는 순백 대신 크림 `#F2EAD8`를 기본으로 쓰고, 보조 텍스트는 뮤트 `#9A968C`를 쓴다.
- 상태와 역할은 둥근 캡슐과 그라데이션을 쓴다. 색만으로 의미를 전달하지 않고 항상 한국어 라벨을 함께 표시한다.
- 주요 액션은 그라데이션 캡슐 버튼으로 표시한다.
- UI 카피는 한국어를 기본으로 한다. `runId`, `workflowId`, 산출물 파일명, CLI 플래그 같은 기술 식별자는 영문을 유지한다.

## 상태 매핑

## 화면

- 앱 셸: 왼쪽 그룹형 사이드바에 `액션`, `작업`, `프로젝트`, `에이전트`, `계정`을 배치한다.
- 대시보드: 실행/승인/프로젝트 요약과 최근 실행, 최근 프로젝트를 표시한다.
- 받은 함: `awaiting-approval` 실행만 모아 승인/거부 흐름으로 연결한다.
- 실행 대시보드: 전체/실행 중/승인 대기/완료 필터, 역할 배지, 실행 카드, `새 실행` 버튼.
- 실행 상세: 티켓형 헤더, 단계 타임라인, 승인 기록, 산출물 목록, 승인/거부/재개/정리 액션.
- 새 실행: 요청 입력, 역할별 자동/사용/끄기 선택, 계획만, 테스트 명령, 최대 수정 횟수.
- 프로젝트 목록: 이름, 로컬/GitHub 소스, 대표 AI, 선택된 AI 배지를 표시한다.
- 새 프로젝트: 이름, 소스(로컬 폴더/GitHub 참조), AI 다중선택, 대표 지정을 단계별로 입력한다.
- 프로젝트 상세: `개요`, `계획`, `조직도`, `실행` 탭을 제공한다.
- 프로젝트 TeamPlan: `계획` 탭에서 개요 입력, 대표 생성, 자유 역할 카드 편집, 담당 AI 선택, 저장 흐름을 제공한다.
- 조직도: `조직도` 탭에서 대표 AI를 정점으로 표시하고, 역할 노드에 역할명, 담당 AI, 상태 라벨을 병기한다.
- 프로젝트 실행: `실행` 탭은 v0.19 실행 엔진 placeholder이며 실제 디스패치는 시작하지 않는다.
- 에이전트: 선택 프로젝트의 AI 목록, 대표 표시, 역할 배정을 보여준다.
- 설정: `baton` 실행 파일 경로. 비워 두면 PATH의 `baton`을 사용한다.

## 수동 QA

- 다크 배경, 크림 텍스트, 그라데이션 캡슐이 대시보드/상세/새 실행/설정에 일관되게 보이는지 확인한다.
- 상태와 역할이 색만이 아니라 한국어 텍스트 라벨로 표시되는지 확인한다.
- 사이드바에 `액션`, `작업`, `프로젝트`, `에이전트`, `계정` 그룹이 표시되는지 확인한다.
- `대시보드`, `받은 함`, `실행`, 프로젝트 행, `AI 조직`, `계정` 선택이 본문 라우팅을 바꾸는지 확인한다.
- `받은 함`이 승인 대기 실행만 표시하고, 승인/거부 버튼이 기존 실행 흐름과 동일하게 동작하는지 확인한다.
- 빈 요청에서는 `시작`이 비활성인지 확인한다.
- 새 실행을 시작하면 `baton run`을 통해 실행되고 목록이 refresh되는지 확인한다.
- `새 프로젝트`에서 빈 이름/소스, 잘못된 GitHub URL, AI 미선택, 복수 AI 대표 미선택이 진행을 막는지 확인한다.
- 로컬 폴더 피커로 프로젝트를 생성하면 `baton project create`를 통해 저장되고 `프로젝트` 목록에 표시되는지 확인한다.
- GitHub URL은 `https://github.com/owner/repo` 참조로만 저장되며 clone이나 네트워크 작업이 없는지 확인한다.
- Codex와 Claude를 함께 선택했을 때 대표 라디오가 나타나고 목록 카드에 `대표` 배지가 표시되는지 확인한다.
- 프로젝트를 선택하면 TeamPlan 화면이 열리고, 개요가 비어 있으면 `대표에게 맡기기`가 비활성인지 확인한다.
- 프로젝트 상세의 `개요`, `계획`, `조직도`, `실행` 탭 전환이 유지되는지 확인한다.
- `조직도` 탭에서 대표 AI가 `👑`과 함께 정점에 표시되고 역할 노드에 역할명, 담당 AI, 상태 텍스트가 보이는지 확인한다.
- TeamPlan이 없는 프로젝트는 조직도 빈 상태 안내를 표시하는지 확인한다.
- `실행` 탭은 v0.19 안내 placeholder만 보여주고 실제 실행을 시작하지 않는지 확인한다.
- `에이전트` 섹션에서 선택 프로젝트의 AI와 대표 표시가 보이는지 확인한다.
- 대표 CLI가 사용 가능할 때 개요 → 생성 → 역할 편집 → 저장 흐름이 동작하는지 확인한다.
- 역할 이름이 비었거나 담당 AI가 프로젝트 AI 밖이면 `저장`이 비활성인지 확인한다.
- 생성/저장 실패 메시지가 화면 하단에 표시되고 앱이 크래시하지 않는지 확인한다.
- 설정에서 잘못된 baton 경로를 저장했을 때 앱이 크래시하지 않고 오류 메시지를 표시하는지 확인한다.
